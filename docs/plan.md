# C/C++ 이론 시험 테스트 사이트 - 구현 계획

## Context

수업 중 수강하는 C/C++ 강의 내용을 바탕으로, 문제은행에서 자동 출제하는 로컬 시험 웹사이트를 구축한다.  
PDF 수업자료는 추후 문제 추가 시 참고 자료로 활용하며, 웹사이트 자체는 순수 HTML/CSS/JS로 구성한다.  
문제는 JSON 파일로 관리하고, 결과는 텍스트 파일로 저장/조회한다.

---

## UI 디자인 방향 (Google Forms 스타일)

Google Forms의 설문 UI를 참고하여 시험 화면을 구성한다.

### 색상 / 배경
| 요소 | 값 |
|------|----|
| 페이지 배경 | `#F0EBF8` (연보라 틴트) |
| 카드 배경 | `#FFFFFF` |
| 헤더 강조색 | `#673AB7` (딥 퍼플) |
| 버튼 주색 | `#673AB7` |
| 버튼 호버 | `#512DA8` |
| 텍스트 기본 | `#202124` |
| 텍스트 보조 | `#5F6368` |

### 레이아웃
- 전체 너비 `max-width: 640px`, 가운데 정렬 (Forms와 동일 기준)
- **헤더 카드**: 상단에 딥 퍼플 두꺼운 보더(`border-top: 10px solid #673AB7`), 시험 제목 + 설명
- **문제 카드**: 흰 카드, `border-radius: 8px`, `box-shadow: 0 2px 6px rgba(0,0,0,0.1)`
- 카드 간 간격 `16px`

### 타이포그래피
- 폰트: `'Google Sans', Roboto, sans-serif` (CDN 없이 시스템 폰트 fallback)
- 문제 번호 + 질문: `16px`, `font-weight: 500`
- 선택지 / 답안: `14px`

### 입력 요소
- 라디오 버튼(O/X, 객관식): 기본 스타일 숨기고 커스텀 원형 선택 UI
- 선택 시 배경 `#EDE7F6` + 보라 테두리
- 주관식 textarea: `border-bottom: 2px solid #673AB7` (Material Input 스타일)

### 진행 표시바
- 카드 상단에 얇은 보라색 progress bar (`height: 4px`)

### 버튼
- 주요 액션(다음/제출): 채워진 보라 버튼, 흰 텍스트, `border-radius: 4px`
- 보조 액션(이전/정답보기): 텍스트 버튼 or 아웃라인 버튼
- 오버라이드 버튼(`✔ 맞음` / `✘ 틀림`): 초록/빨강 아웃라인, 선택 시 채워짐

```

├── docs/

│   └── subjective.json          ← 주관식 문제 목록

└── app.js                       ← 문제 로딩, 출제, 채점 로직
```

## 시험 구성 (20문제 고정)

| 유형 | 비율 | 문제 수 | 배점 | 자동채점 |
|------|------|---------|------|---------|
| O/X | 20% | 4문제 | 각 5점 | O |
| 객관식 5지선다 | 30% | 6문제 | 각 5점 | O |
| 주관식 (코드 빈칸, 개념, 실행결과) | 50% | 10문제 | 각 5점 | 모범답안 비교 (자가채점) |

- **총점: 100점** (문제당 균등 5점)
- **합격 기준: 12문제 이상 정답 = 60점 이상 → PASS** (11문제 이하 = FAIL)

## JSON 문제 형식

### ox.json
```json
[
  {
    "id": "ox_001",
    "type": "ox",
    "question": "C언어에서 배열의 인덱스는 0부터 시작한다.",
    "answer": true,
    "explanation": "C/C++ 배열은 0-indexed이다."
  }
]
```

### multiple.json
```json
[
  {
    "id": "mc_001",
    "type": "multiple",
    "question": "다음 중 C언어 기본 자료형이 아닌 것은?",
    "options": ["int", "float", "string", "char", "double"],
    "answer": 2,
    "explanation": "string은 C++ STL에서 제공하는 클래스이다."
  }
]
```

### subjective.json
```json
[
  {
    "id": "sub_001",
    "type": "subjective",
    "subtype": "code_blank",
    "question": "다음 코드의 빈칸을 채우시오.",
    "code": "int arr[5] = {1,2,3,4,5};\nprintf(\"%d\", arr[___]);  // 3 출력",
    "answer": "2",
    "explanation": "배열 인덱스 2는 세 번째 요소(값 3)를 가리킨다."
  }
]
```

## 문제 풀이 UX 상세

### 정답 공개 기능 (모든 문제 공통)
- 각 문제마다 「정답 보기」 토글 버튼 존재
- 기본 상태: 정답/해설 숨김
- 버튼 클릭 시: 정답 + 해설 표시, 버튼 텍스트 → 「정답 숨기기」

### 사용자 정오답 오버라이드 버튼 (모든 문제 공통)
- 모든 문제에 「✔ 맞음」 / 「✘ 틀림」 버튼 쌍이 표시됨
- 클릭 시 해당 문제의 채점 결과를 사용자가 직접 재지정
- **주관식**: 오버라이드 버튼 클릭이 **필수** → 누르기 전엔 다음 문제로 이동 불가
- **O/X, 객관식**: 오버라이드 버튼은 **선택** → 누르지 않아도 다음 문제로 이동 가능

### 채점 흐름 (20문제 완료 후)
1. **기본 채점**: O/X·객관식은 자동 채점, 주관식은 사용자가 오버라이드로 입력한 값 사용
2. **오버라이드 반영**: 사용자가 별도로 지정한 정오답이 있으면 자동 채점 결과를 덮어씀
3. **최종 점수 계산**: 오버라이드 반영 후 최종 점수 확정

## 페이지별 기능

### index.html (시작 화면)
- 사이트 제목 표시
- 「시험 시작」 버튼 → exam.html로 이동
- 「결과 보기」 버튼 → result.html로 이동

### exam.html (시험 진행)
- `fetch('./questions/ox.json')` 등으로 3종 JSON 로드
- 문제은행에서 무작위 추출: O/X 4개, 객관식 6개, 주관식 10개
- 문제 섞어서 1~20번 순서로 표시
- 진행 표시바 (몇 번째 문제 / 20)
- 각 문제 하단 UI 구성:
  - 답안 입력 영역 (유형별)
  - 「정답 보기/숨기기」 토글 버튼
  - 「✔ 맞음」 / 「✘ 틀림」 오버라이드 버튼
  - 이전 / 다음 버튼 (주관식은 오버라이드 선택 전 다음 비활성화)
- 20번 문제에서 「제출」 버튼 → sessionStorage에 답안+오버라이드 저장 후 result.html로 이동

### result.html (결과/채점)
- **1단계 자동채점 표시**: O/X·객관식 자동 채점 결과 보여줌
- **2단계 오버라이드 반영**: 사용자가 지정한 오버라이드 적용 후 최종 점수 재계산
- 문제별 상세: 내 답 / 정답 / 오버라이드 여부 / 해설 표시
- 점수 요약: 유형별 정답 수 + 총점(문제당 5점, 100점 만점) + **PASS / FAIL 판정** (12문제↑ PASS)
- 「결과 저장 (.txt)」 버튼: N회차 시험결과로 Blob 다운로드
- 「이전 결과 불러오기」 버튼: `<input type="file">` 로 .txt 선택 → 내용 표시

## 결과 텍스트 파일 형식

```
===== C/C++ 이론 시험 결과 (3회차) =====
날짜: 2026-04-08 14:30:00
총점: 72 / 100  |  판정: FAIL  (합격 기준: 60점 / 12문제)

[O/X]   3/4 정답  (15점)
[객관식] 5/6 정답  (25점)
[주관식] 자가채점 6/10 (30점)
정답 문제: 14 / 20

--- 문제별 상세 ---
1. [OX] 맞음 | 내 답: O | 정답: O
   Q. C언어에서 배열의 인덱스는 0부터 시작한다.
   해설: C/C++ 배열은 0-indexed이다.

2. [OX] 틀림(오버라이드) | 내 답: O | 정답: X
   Q. ...
   해설: ...
...
```

- 회차는 `results/` 디렉터리 내 파일 수 + 1로 자동 계산 (또는 수동 입력)

## 실행 방법

> ⚠️ `fetch()`로 로컬 JSON을 읽으려면 로컬 서버가 필요합니다 (file:// 프로토콜은 CORS 차단).

```bash
cd veda테스트준비
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

또는 VS Code **Live Server** 확장 사용.

## 구현 순서

1. 프로젝트 디렉터리 및 기본 파일 생성
2. `questions/` 에 각 JSON 샘플 문제 10개씩 작성 (추후 PDF 기반 추가)
3. `style.css` - 공통 스타일 (심플 다크 테마)
4. `app.js` - JSON 로드, 문제 랜덤 추출, 채점/오버라이드/결과 저장 로직
5. `index.html` - 시작 화면
6. `exam.html` - 시험 진행 UI (정답 보기 토글, 오버라이드 버튼 포함)
7. `result.html` - 채점 2단계(자동→오버라이드 반영), 결과 저장/불러오기 UI
8. 통합 테스트 (로컬 서버 기동 후 전체 흐름 확인)
